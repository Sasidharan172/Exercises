CSS Styling Methods Demonstration

External CSS
External CSS Example
This box is styled using external CSS (simulated using data URL)
/* external-styles.css */ .external-css { background: #e74c3c; color: white; padding: 20px; margin: 20px 0; border-radius: 8px; }
Internal CSS
Internal CSS Example
This box is styled using internal CSS (defined in style tag)
<style> .internal-css { background: #3498db; color: white; padding: 20px; margin: 20px 0; border-radius: 8px; } </style>
Inline CSS
Inline CSS Example
This box is styled using inline CSS (style attribute)
<div style="background: #2ecc71; color: white; padding: 20px; margin: 20px 0; border-radius: 8px;"> This box is styled using inline CSS </div>
CSS Method Pros Cons Best Used For
External CSS - Separation of concerns
- Better caching
- Easy maintenance
- Reusable across pages 		- Additional HTTP request
- May delay page rendering 		- Large websites
- Multiple pages
- Consistent styling
Internal CSS - No external files
- Page-specific styles
- Quick to implement 		- Increases page size
- Not reusable across pages
- Mixing content with styles 		- Single-page websites
- Quick prototypes
- Page-specific styles
Inline CSS - Highest specificity
- Quick fixes
- Element-specific styling 		- Hard to maintain
- Mixes content with styles
- No reusability 		- Quick fixes
- Email templates
- Dynamic styles
